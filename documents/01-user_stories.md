# 系統使用者故事 (User Stories)

## 一、 角色定義
*   **保全人員 (Security Guard)**：第一線執勤人員，負責簽到、巡檢與排班意向提交。
*   **主管/調度員 (Manager)**：負責人力調度、假單審核、即時勤務監控與正式班表發布。
*   **會計 (Accountant)**：負責薪資結算、工時統計與報表匯出。
*   **系統管理員 (Admin)**：負責站點設定、權限管理、系統技術維護與基礎資料設定。

## 二、 使用者故事內容

### 1. 保全人員 (Security Guard)
*   **出勤管理**
    *   【簽到/退】我可以在到達崗位後，透過 LINE LIFF 掃描崗位專屬 QR Code，系統會自動比對 GPS 定位（地理圍欄），確認位置一致後完成簽到/退，確保執勤真實性。
    *   【考勤檢視】我可以隨時在手機上查看自己當月每一天的出勤紀錄（含遲到、早退標記）。
    *   【入帳通知】當薪資發放後，我可以在 LINE 上收到即時的入帳明細通知。
*   **排班與假務**
    *   【預約排班】在每月最後一週，我可以提交下個月的偏好班表或休假意向。
    *   【查看班表】我可以隨時查看自己未來一週或一個月的確定班表，並收到排班更新通知。
    *   【請假申請】我可以線上提交假單（病、事、特假等），並上傳相關證明圖片。
    *   【調換班請求】我可以與同事發起換班請求，在雙方確認且主管核准後系統自動更新班表。
*   **日常作業**
    *   【巡邏打卡】我可以依照規定的巡邏點掃描 QR Code 並拍照回報，系統自動記錄時間與位置。
    *   【緊急回報】遇到突發狀況時，我可以使用緊急按鈕即時通知主管並傳送當前座標。

### 2. 主管/調度員 (Manager)
*   **人力與調度**
    *   【即時監控】我可以透過管理介面即時監控所有站點的到勤狀態，若有人逾時未簽到，系統應發出異常告警。
    *   【機動調度】我可以隨時查詢目前「待命/非輪值」的機動保全清單，快速指派人員支援缺勤崗位。
    *   【排班管理】系統應能根據保全意向與站點需求「自動生成建議班表」，我可以在此基礎上進行人工微調並發布。
*   **審核與統計**
    *   【線上核假】我可以即時收到請假申請並進行審核，同時系統會提示該時段是否產生人力缺口。
    *   【統計報表】我可以查詢各站點、各人員的每月出勤率、遲到次數與異常紀錄，作為管理依據。

### 3. 會計 (Accountant)
*   **薪資與結算**
    *   【工時結算】系統應根據簽到紀錄、核准假單與排班表，自動結算每位人員的總工時、加班、夜班津貼。
    *   【薪資明細】我可以設定基本時薪、加給標準與勞健保扣款，系統自動產出應發金額。
    *   【報表匯出】我可以一鍵產生全公司的月薪資總表（Excel 格式），用於銀行匯款與財務存檔。
    *   【獎懲調整】我可以手動增加特定人員的獎金或扣除違規罰款，並記錄原因。

### 4. 系統管理員 (Admin)
*   **系統設定與維護**
    *   【權限控管】我可以設定不同管理員（主管、會計）的存取權限。
    *   【基礎資料維護】管理站點資訊（座標、半徑、班別定義）與員工基本資料。
    *   【技術環境管理】確保系統後端與 LINE API 串接正常，監控自動排班排程執行狀況。

## 三、 技術堆疊與規範 (Technical Stack & Requirements)
1.  **後端框架**：採用 **Laravel 12 (最新穩定版本)**，充分利用其 PHP 8.3/8.4 新特性與類型安全。
2.  **管理介面開發**：採用 **Laravel Livewire v3 + Tailwind CSS** 自製專用後台，確保與 Laravel 12 完全相容，並提供主管與會計彈性的網頁操作與報表功能。
3.  **前端/手機端介面**：採用 **LINE 官方帳號 + LIFF (LINE Front-end Framework)**。
    *   頁面邏輯建議使用 **Laravel Livewire**，以 PHP 為核心開發類 SPA 的互動體驗，適合 LINE 內部載入。
4.  **定位防弊**：簽到功能強制開啟 **GPS 地理圍欄 (Geofencing)** 驗證，並配合 QR Code 掃描確保人員在場。
5.  **通知機制**：重要訊息（排班異動、異常告警、薪資入帳）透過 **LINE Messaging API** 進行 Push Message 即時推播。
6.  **自動排班邏輯**：系統需支援基本排班規則（如：勞基法工時規範、每人每月休假天數、站點最低人力需求），支援自動生成與人工微調。
7.  **報表格式**：會計匯出功能需符合台灣常見之 Excel 薪資明細格式。
